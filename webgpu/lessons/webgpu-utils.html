WebGPU Utils and wgpu-matrix

What you should take away from this article

Using WebGPU is very verbose. So verbose that it gets easier to understand if you use some helpers so that you can concentrate on the higher level concepts.

For example, say you were learning math. Your teacher teaches you what ‚Äúaverage‚Äù means and how to compute the average of some set of numbers. Once they‚Äôve taught you, they then move on to other things and just say ‚Äúhere you compute the average‚Äù. For example:

To compute the standard deviation

  • Calculate the Average of all your data
  • For each number in your data set, calculate the difference between that number and the average.
  • After finding each difference, square it.
  • Take the Square root of the average the squared differences

They don‚Äôt re-explain how to calculate an average. You‚Äôve already learned it and they can now just refer to what you‚Äôve already learned

Similarly in WebGPU we have the concept of creating structures for uniforms in WGSL. Then creating one or more uniform buffers, filling those buffers with data using TypedArrays. We‚Äôve covered this extensively in the first 20-30 articles on this site and in the article on memory layout.

At some point though, it becomes harder to understand the code dealing with these details instead of just saying ‚Äúset the uniform‚Äù and you, having learned previously that ‚Äúset the uniforms‚Äù means "compute the offset to the various pieces of data, make typed arrays views to make it possible to set that data. And then later, before rendering, set it and upload the values to the GPU.

As such, don‚Äôt be afraid of the libraries used on this site. Almost all of their functionality is explained extensively in the first articles on the site. Some more details are provided below.

Many of the examples on this site use two libraries.

wgpu-matrix

The first is wgpu-matrix. wgpu-matrix is a collection of the same functions we wrote in the article on matrix math through the article on perspective projection as well as the article on lighting.

There‚Äôs nothing special happening here. If you want to know how any of the math functions work you can go read the articles listed above.

The second is webgpu-utils.

WebGPU Utils is a collection of the other useful functions we‚Äôve written in various articles. For example, the functions

  • loadImageBitmap

All of which we created in the article on importing textures.

And it includes how we added support for premultipliedAlpha in the article on transparency and blending.

The library also includes

from the article on environment maps.

I hope you can see, THIS VERBOSITY IS DISTRACTING FROM THE POINT OF THE ARTICLES! All we really wanted to say is ‚Äúchange your WGSL structure to this, then set the values before drawing‚Äù but instead we have 40+ lines of code changes to show PER EXAMPLE.

Using the makeShaderDataDefinitions and makeStructuredView, all the JavaScript above can be changed to these 7 lines.

const defs = makeShaderDataDefinitions(code);
const uni = makeStructuredView(defs.uniforms.uni);

const uniformBuffer = device.createBuffer({
  size: uni.arrayBuffer.byteLength,
  usage: GPUBufferUsage.UNIFORM | GPUBufferUsage.COPY_DST,
});

That‚Äôs it. Between samples, we‚Äôd change our structure as appropriate but these 2 functions would create all of those offsets and views for us.

these 2 lines

Make a ‚Äústructured view‚Äù for uni, the uniform binding we defined in our WGSL.

Effectively those lines make this

const arrayBuffer = new ArrayBuffer(256);
const uni = {
  arrayBuffer,
  set: function(data) { /* helper */ },
  views: {
    normalMatrix: new Float32Array(arrayBuffer, 0, 12),
    worldViewProjection: new Float32Array(arrayBuffer, 48, 16),
    world: new Float32Array(arrayBuffer, 112, 16),
    color: new Float32Array(arrayBuffer, 176, 4),
    lightWorldPosition: new Float32Array(arrayBuffer, 192, 3),
    viewWorldPosition: new Float32Array(arrayBuffer, 208, 3),
    shininess: new Float32Array(arrayBuffer, 220, 1),
    lightDirection: new Float32Array(arrayBuffer, 224, 3),
    innerLimit: new Float32Array(arrayBuffer, 236, 1),
    outerLimit: new Float32Array(arrayBuffer, 240, 1),
  },
};

There‚Äôs no magic here, except maybe the fact that makeShaderDataDefinitions actually parses the WGSL to pull out enough data to make these views.
